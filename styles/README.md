# Styles

Shared design tokens, CSS variables, and theme files used across projects.

Projects are not required to use anything here — this is a convenience layer, not a constraint. If a project has a distinct visual identity that conflicts with shared tokens, it should manage its own styles.

What belongs here:
- CSS custom properties (design tokens)
- Base resets or typography defaults
- Color palettes or theme definitions

What does not belong here:
- Component-level styles (those live in the project)
- Framework-specific configurations (those live in the project)
