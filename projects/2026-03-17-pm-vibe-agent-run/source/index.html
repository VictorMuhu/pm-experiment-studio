How this works

            
A run log is only useful if it’s consistent. This prototype enforces the shape.

          

          
        


        

          

            
Flow

            
    
              
  • Paste whatever you captured today (Slack snippets, meeting bullets, open questions).
    • 
              
  • Parse to split notes into risks, decisions, asks, and progress updates.
    • 
              
  • Triage by resolving items or converting fuzzy notes into explicit asks.
    • 
              
  • Copy an exec-ready update with a stable structure and daily “vibe” score.
    • 
            

          


          

            
Vibe scoring (prototype)

            
A simple heuristic: open high-severity risks and unconfirmed decisions pull the score down; resolved items and shipped updates lift it. It’s meant to be comparable day-to-day, not “correct.”

          


          

            
Tip
