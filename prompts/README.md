# Prompts

Reusable LLM prompts for project generation, review, and iteration.

Naming convention: `[verb]-[object].md` — e.g., `generate-problem-statement.md`, `review-decision-log.md`, `suggest-scope-cuts.md`.

Each prompt file should open with a comment block explaining:
- What the prompt does
- What context it needs
- What output to expect

---

Prompts here are not magic. They're starting points. Treat the output as a draft, not a deliverable.
