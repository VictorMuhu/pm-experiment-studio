# weekly-builds/

This folder holds structured weekly builds — short, focused experiments scoped to a single week.

## Purpose

Weekly builds are the bridge between rough scratch work and more complete projects. Each build should tackle one small concept, system component, or product idea and be wrapped up within a week.

## Folder Structure

Each week gets its own subfolder:

```
weekly-builds/
├── 2026-W01-rag-retrieval-prototype/
│   ├── README.md
│   ├── code/
│   └── notes.md (optional)
├── 2026-W02-eval-rubric-experiment/
│   ├── README.md
│   ├── code/
│   └── notes.md (optional)
```

## Guidelines

- One subfolder per week
- Name folders using the format: `YYYY-WXX-short-description`
- Each build should include at minimum a `README.md` explaining what was built and what was learned
- Keep builds small and focused — one concept at a time
- A build that grows beyond one week's scope belongs in `projects/`

## Recommended Contents for Each Weekly Build

- `README.md` — what it is, what was explored, what was learned
- `code/` — implementation files
- `notes.md` (optional) — raw observations, open questions, next steps
