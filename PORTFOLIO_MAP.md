# PM AI Portfolio Map

## Purpose

This file is the control center for the repository.

It helps track:
- what is being explored
- what is in active development
- what is ready to be promoted
- what has already been promoted to the demo layer

This repository is the experimentation lab, not the polished showcase.

---

## Repository Role

`pm-experiment-studio` is the working lab for:
- fast experiments
- weekly builds
- rough prototypes
- evolving product ideas
- technical investigations

It feeds stronger, polished work into:
- `pm-ai-demo`

---

## Workflow

scratch → weekly-builds → projects → pm-ai-demo

### Meaning of each stage

- **scratch**
  - raw exploration
  - messy and fast
  - useful for trying ideas quickly

- **weekly-builds**
  - structured short builds
  - one small concept or system step at a time

- **projects**
  - larger efforts with multiple components
  - more complete and more reusable
  - still evolving

- **pm-ai-demo**
  - only polished, demo-ready, externally presentable projects

## Promotion Criteria

A project should move to `pm-ai-demo` only when it has:

- a working system
- a clean README
- architecture documentation
- AI flow documentation
- evaluation or quality bar
- clear tradeoffs and failure modes
- a Loom demo link
- a stable demo path

If a project does not meet this bar, it stays in this repo.
